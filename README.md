# Catalin Editorial Tech Portfolio

Rebuilt from scratch as a React + TypeScript + Vite app.

## Run

```bash
npm install
npm run dev
```

## Build

```bash
npm run build
npm run preview
```

`npm run build` refreshes `src/content/evidence.json` before bundling.

## Notes

- Content is in `src/content/*.json`.
- Work entries support `liveUrl` and `stack` fields in `src/content/work.json`.
- Bot evidence snapshot is generated by `npm run evidence:refresh`.
- Optional: set `BOT_REPO_PATH` if your bot repo is outside the default sibling location.
- Main routes:
  - `/`
  - `/playground`
  - `/work/:slug`
  - `/lab-log`
  - `/contact`

## Analytics (Plausible, optional)

1. Copy `.env.example` to `.env`.
2. Set `VITE_SITE_URL` to your deployed domain.
3. Set `VITE_PLAUSIBLE_DOMAIN` to your deployed domain if using Plausible.
4. Set `VITE_FORMSPREE_ENDPOINT` to your Formspree form endpoint.
5. Start or build the app as normal.

If `VITE_PLAUSIBLE_DOMAIN` is unset, analytics stays disabled.

Important:
- This app now supports runtime config via `public/runtime-config.js` generation in Docker startup.
- On Railway, set `VITE_*` vars in service variables and redeploy/restart service to apply.

## Deploy

- Railway (recommended for your setup):
  1. Create a new Railway service from this repo.
  2. Select Dockerfile deploy (this repo includes a production-ready Dockerfile).
3. Set env vars in Railway:
   - `VITE_SITE_URL`
   - `VITE_FORMSPREE_ENDPOINT`
   - `VITE_PLAUSIBLE_DOMAIN` (optional)
   - `VITE_PLAUSIBLE_SRC` (optional)
   - `VITE_BOT_URL` (optional, defaults to current bot service URL in code)
   - `VITE_BOT_WS_URL` (optional, defaults to current bot service WS URL in code)
4. Attach your custom domain.

- Netlify config is pre-added in `netlify.toml`.
- Vercel SPA rewrite config is pre-added in `vercel.json`.
